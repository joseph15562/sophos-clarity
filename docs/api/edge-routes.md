# Supabase Edge Functions — route map

Deployed functions live under `https://<project>.supabase.co/functions/v1/<name>`.  
The main REST surface for the web app is **`api`** (JWT gateway) and **`api-public`** (shared links, guest upload). **`api-agent`** uses `X-API-Key` (connector).

| Function                   | Auth                                               | Purpose                                                                                                                                                                                                                                               |
| -------------------------- | -------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **api**                    | JWT (Supabase gateway) + route-specific exceptions | Monolithic router: assessments, firewalls, admin, agents, passkeys, portal viewers, SE health/config-upload, service keys, PSA integrations, send-report. See comments in [`supabase/functions/api/index.ts`](../../supabase/functions/api/index.ts). |
| **api-public**             | Token paths / anon where coded                     | Shared reports, guest config upload flows, unauthenticated entry points split from `api`.                                                                                                                                                             |
| **api-agent**              | `X-API-Key` → `authenticateAgent`                  | Connector: config, firewalls, heartbeat, submit, verify-identity.                                                                                                                                                                                     |
| **parse-config**           | User JWT (function logic)                          | AI-assisted config parsing (Gemini); CORS-enforced.                                                                                                                                                                                                   |
| **portal-data**            | Varies by path                                     | Read-only portal JSON for client portal UI (`GET`, CORS).                                                                                                                                                                                             |
| **sophos-central**         | JWT / guest publishable key paths                  | Browser proxy to Sophos Central APIs.                                                                                                                                                                                                                 |
| **send-scheduled-reports** | Cron / user JWT                                    | Enqueues due reports into **`job_outbox`** (producer); preview still builds email via JWT POST.                                                                                                                                                       |
| **process-job-outbox**     | Cron (`CRON_SECRET` Bearer when set)               | Claims **`job_outbox`** jobs, sends via Resend, updates **`scheduled_reports`**.                                                                                                                                                                      |
| **agent-nudge**            | Service role / cron                                | Queues assessment nudges; marks stale agents offline.                                                                                                                                                                                                 |
| **agent-daily-presence**   | Service role / cron                                | Writes `agent_daily_presence` (UTC day per agent) for the fleet Agent Status (7-day) timeline.                                                                                                                                                        |
| **regulatory-scanner**     | Service role / cron                                | RSS fetch + optional Gemini summarise + `regulatory_updates` upsert.                                                                                                                                                                                  |

In-app REST reference (subset): **API hub** in the product (`/api` → API Explorer) and [`ApiDocumentation.tsx`](../../src/components/ApiDocumentation.tsx).

**OpenAPI (evolving):** [`openapi.yaml`](openapi.yaml) — JWT **`api`** routes, selected **`api-public`** paths (`GET /shared/{token}`, `GET /shared-health-check/{token}`), **`portal-data`**, **`parse-config`**, and **cron/proxy** path entries (`send-scheduled-reports`, `process-job-outbox`, `agent-nudge`, `agent-daily-presence`, `regulatory-scanner`, `sophos-central`) with path-level server URLs. Zod-validated bodies for **`api`** live in [`supabase/functions/_shared/api-schemas.ts`](../../supabase/functions/_shared/api-schemas.ts); keep YAML aligned when routes change.
