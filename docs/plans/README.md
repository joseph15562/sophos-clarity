# Plans (versioned)

Product and technical **plans** agreed in Cursor live here so they are **tracked in git** and visible on **GitHub**, not only in editor-local storage.

**Convention:** add or update a `.md` file in this folder when a plan is finalized or changed; name files by topic (kebab-case).

**Backfill:** Files named `*_xxxxxxxx.md` (8-char hex suffix) were copied from local Cursor plan storage (`~/.cursor/plans/*.plan.md`) so history stays in-repo. New plans can use shorter names if you prefer.

See also: `.cursor/rules/plans-in-git-repo.mdc`.
