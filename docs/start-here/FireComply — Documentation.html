MSP Workflows & Best Practices

Six core workflows that MSPs use daily, weekly, and quarterly — with best practices for each.

Workflow 1: New Customer Onboarding

Scenario: You've signed a new managed firewall customer. You need a baseline assessment, compliance posture, and a professional deliverable within the first week.
  1. Connect Sophos Central — discovers the customer's tenant and firewalls
  2. Install a Connector Agent on a machine with network access to the firewall(s)
  3. Set customer context — environment type, country, compliance frameworks
  4. Run the full assessment — deterministic analysis + AI reports
  5. Save to Report Centre — the customer's baseline is archived
  6. Configure the Client Portal — brand it, set the vanity slug, add portal viewers
  7. Set up scheduled reports — monthly or quarterly, auto-delivered
Tip: Do the first assessment before the onboarding call. Walking through a live, populated portal on the kick-off call makes a strong first impression.

Workflow 2: Quarterly Business Review (QBR)

Scenario: It's QBR season. You need to present security posture, compliance status, and improvement trajectory to 15 customers in two weeks.
  1. Open Portfolio Insights → set range to 90 days → screenshot the portfolio trends
  2. For each customer: review score trend, note open findings, pull the latest report
  3. On the QBR call: share the Client Portal on screen — walk through score, history, findings
  4. Reference improvement: "Your score has gone from 54% to 78% since we started"
  5. After the call — email the report from Report Centre with one click
Tip: Use Portfolio Pulse in Customer Directory for the executive stat — "72% of our managed estate is at Grade A or B" is a powerful number for boards.

Workflow 3: Responding to a Security Incident

Scenario: A customer reports suspicious activity. You need to check if their firewall config contributed to the exposure.
  1. Open Mission Control — check recent alerts for this customer
  2. Open Fleet Command → search for customer → review status and score
  3. Open Drift Monitor → check recent snapshots: did the config change? Were rules added or removed?
  4. Open the latest assessment → review findings by severity
  5. Generate a focused report if needed — the Technical Handover documents the config state at assessment time
Important: Drift Monitor is your timeline. When the customer says "we didn't change anything", the snapshot history tells the truth.

Workflow 4: Compliance Audit Preparation

Scenario: A customer's compliance officer needs evidence that their firewall config meets Cyber Essentials / GDPR / ISO 27001 requirements.
  1. Load the customer's config in the Assess Workbench
  2. Set the correct compliance frameworks
  3. Review deterministic compliance alignment
  4. Generate the Compliance Readiness Report — control-by-control evidence tables with pass/partial/not met status
  5. Share via Client Portal, direct email, or shared link
Tip: Run a fresh assessment before the audit. Stale data undermines credibility. Use the Connector Agent's "scan now" capability.

Workflow 5: Upsell & Expansion

Scenario: You want to identify upsell opportunities across your customer base.

Common upsell signals from the deterministic analysis:

Gap FoundUpsell Conversation
No DPI (SSL/TLS inspection)Xstream Protection
No web filtering on WAN rulesWeb Protection licensing
No Sophos endpoints (Heartbeat)Endpoint opportunity
No MDRMDR upsell
No DNS ProtectionDNS product conversation
Tip: Use the Remediation Impact Simulator on a call: "If we enable DPI and web filtering, your score goes from 52% to 71% — that's the impact of Xstream Protection."

Workflow 6: Team Collaboration & Handoffs

Scenario: An engineer on your team is leaving, or you're sharing workload across a team.
  • Everything is in the workspace — no data lives on individual machines (except Connector Agent installs)
  • Add team members via Management Panel → Team: Admin (full access) or Viewer (read-only)
  • All assessments, reports, and customer data are shared across the workspace
  • Client Portal configurations are org-wide — any team member can manage them

General MSP Best Practices

  1. Connect Sophos Central first, install agents second. Central gives you instant fleet visibility; agents give you config analysis.
  2. Set up Client Portals for every customer. Even if the customer never logs in, having a branded portal ready for QBRs saves time.
  3. Use scheduled reports as your retention tool. Regular, automated deliverables remind customers why they pay you.
  4. Lead with the deterministic layer. AI generates the reports, but the score, findings, and compliance mapping are computed locally — same config always produces the same result.
  5. Anonymisation is your privacy story. Data is anonymised before it leaves the browser. The AI never sees real hostnames or IPs.
  6. Check the Changelog (/changelog) before customer meetings.
  7. Use the in-app guided tours when onboarding team members.

Manual Upload vs Connector Agent

Manual Upload

  • Someone remembers to export the config
  • Config sits on someone's desktop
  • Assessments go stale between reviews
  • One firewall at a time
  • No drift detection

Connector Agent

  • Automated on a schedule (daily, weekly)
  • Results push directly to FireComply
  • Fleet and Mission Control update automatically
  • Monitor dozens of firewalls per agent
  • Drift Monitor catches changes between scans

  1. Open SE Health Check → click "Request Config Upload"
  2. Customer receives a branded email with instructions and a secure upload link
  3. Customer uploads and optionally connects Central
  4. You get notified → load the config → set the licence tier
  5. Walk through qualifying questions on the call — each toggle adjusts the score live
  6. Click "Send to customer" to email the PDF before the call ends
Tip: Set the licence tier before the call. The score changes significantly between Standard and Xstream.

Workflow 2: Post-Sales Improvement Review

Scenario: You sold Xstream Protection 6 months ago. Time to show the customer their improvement.
  1. Open the customer's previous health check
  2. Request a new config upload (or use connector)
  3. Walk through the score improvement — "Last time 52% Grade D, now 78% Grade B"
  4. Highlight findings resolved and any new issues
  5. Send the updated report
Tip: Reference the previous report. "We flagged 4 fully open rules — you've resolved 3." Use this as a renewal tool.

Workflow 3: Quick Internal Demo

Scenario: You need to show a colleague or partner what FireComply does in 2 minutes.
  1. Go to the login page → click "Try Demo Mode"
  2. Walk through: Hero outcome panel → Security Posture Scorecard → Remediation Impact Simulator → Generate a report
  3. If time: show Fleet, Customers, Mission Control with demo data
Tip: Lead with the outcome, not the upload. Use the 2-minute demo script for structure.

General Best Practices

  1. Use the in-app guided tours when showing FireComply to someone for the first time
  2. The deterministic layer is your credibility. Findings and scores are computed locally — lead with this
  3. Never share raw config data. FireComply anonymises before AI processing. Mention this to security-conscious customers
  4. Use the Remediation Impact Simulator on calls — turns a static report into an interactive conversation
  5. Check the Changelog (/changelog) before customer meetings
SE Health Check

Using FireComply with a Customer

Before, during, and after the meeting — with customer FAQ answers you can use verbatim.

Before the Meeting

  • Know the customer's environment type (Education, Healthcare, Finance, etc.)
  • Know their country and any specific compliance requirements
  • Check if they have Sophos Central — have them prepare API credentials
  • Decide whether to request config upload in advance (recommended) or upload live
  • If pre-sales: know which licence tier they're evaluating

During the Meeting

Opening (2 minutes)

Share your screen showing the SE Health Check results. Start with the best-practice score: "Your firewall scored 66% against Sophos best practices, which is a Grade C." Frame positively: "Out of 37 checks, 22 passed."

Walking Through Findings (10–15 minutes)

Use the category view: Device Hardening, Visibility & Monitoring, Encryption & Inspection, etc. For each failed check: what it looks for, what it found, the Sophos recommendation, and how to remediate.

Use qualifying questions as natural discussion points:

  • "I see Security Heartbeat isn't configured — do you have Sophos-managed endpoints?" → sales opportunity
  • "MDR threat feeds aren't in the export — is that enabled?" → MDR conversation
  • "No SSL/TLS decrypt rules — is there a reason DPI isn't active?" → DPI discussion

Adjusting the Score Live (2 minutes)

Toggle qualifying questions — the score recalculates in real-time. "If we acknowledge MDR is configured and exclude Guest zone from DPI, your score goes from 66% to 74%."

Closing (2 minutes)

Click "Send to customer" — the branded PDF arrives during the call. Or share a read-only link they can bookmark.

Common Customer Questions

"Is my config data safe?"

Yes. The config is parsed client-side in your browser. When AI reports are generated, the data is anonymised — hostnames and IPs are replaced with tokens before leaving the browser. No raw config is stored server-side.

"Do I need a FireComply account?"

No. The upload portal works with just the secure link — no account required. The SE handles everything on their end.

"How does the score work?"

The score is based on 37 Sophos best-practice checks, weighted by category and severity. It's deterministic — same config always produces the same score. The AI is only used for the written report, not the score itself.

"What firewalls are supported?"

Sophos XGS and SFOS (XG) firewalls. Both HTML exports (from the web console) and XML entities exports (from backup or API) are supported.

The Full Workflow

1. Request a Customer Config Upload

  1. Click "Request Config Upload"
  2. Fill in: Customer name, Contact person, Email address, Link expiry (1–30 days)
  3. Click Send — the customer receives a branded email with instructions, a drag-and-drop upload portal, and optional Central API connection

2. Customer Uploads (Their Side)

  1. They receive a professional email from FireComply
  2. They click the link and see a branded upload page
  3. They drag and drop their entities.xml file
  4. Optionally enter Sophos Central API credentials for enrichment
  5. Done — no account needed

3. SE Gets Notified

You receive an email: "Configuration Uploaded" with an "Open FireComply" button. The config appears in your Upload Requests panel.

4. Load and Review

Click "Load Config". Set the licence tier (Xstream Protection, Standard, or Individual Modules). Everything pre-populates.

6. Review the Score

Template-generated (not AI) — the numbers are always mathematically accurate. Read them aloud on the call or include in the report.
