# Client data layer (TanStack Query vs direct Supabase)

**Policy:** Prefer **`useQuery` / `useMutation`** with stable keys from [`src/hooks/queries/keys.ts`](../../src/hooks/queries/keys.ts) for server-backed reads and writes. Pass **`signal`** into `fetch` when using `useQuery` so navigation unmounts cancel in-flight requests.

**Boundary (ADR 0004):** Domain **`supabase.from` / `supabase.rpc`** and raw Edge **`fetch`** for reads/writes belong in **`src/lib/data/*`** and **`src/hooks/queries/*`**, not in **`src/components`**, **`src/pages`**, or route modules — except for documented exceptions below. See [`docs/adr/0004-frontend-data-boundary.md`](../adr/0004-frontend-data-boundary.md).

## Intentional exceptions (direct Supabase / `fetch`)

| Area                            | Why not Query (yet)                                                                                                                                                                                                                                                       |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Auth session**                | `supabase.auth.getSession()` is synchronous-ish orchestration; not cached as entity lists.                                                                                                                                                                                |
| **Large wizards / mega-panels** | SE Health Check inner state: **send-report** `fetch` uses **`AbortSignal`**; **prepared-by migration** uses **`supabaseWithAbort`**. Remaining **`supabase.from`** on save/rename paths is intentionally inline until a single **`se_health_checks`** data module exists. |
| **One-off mutations**           | Simple `.update` / `.delete` where invalidation is wired manually — prefer wrapping in `useMutation` when touching lists shown elsewhere.                                                                                                                                 |

**Migrated:** **PortalConfigurator** uses **`usePortalTenantBootstrapQuery`** + **`usePortalConfigSaveMutation`** (`src/lib/data/portal-tenant-bootstrap.ts`, **`portal-config-save.ts`**). **ScheduledReportSettings** uses **`useScheduledReportsQuery`** and **`useScheduledReport*Mutation`** hooks. **InviteStaff** uses **`useOrgTeamRosterQuery`** plus **`useOrgInviteMutation`**, **`useOrgInviteRevokeMutation`**, **`useOrgMemberRemoveMutation`**. **PasskeyManager** delete uses **`usePasskeyDeleteMutation`**. **MspSetupChecklist** uses **`useMspSetupStatusQuery`** (`src/lib/data/msp-setup-status.ts`). **RemediationPlaybooks** / **PlaybookLibrary** write **`remediation_status`** via **`useRemediationDeltaMutation`** / **`useRemediationPlaybookToggleMutation`** (`src/lib/data/remediation-status.ts`). **AgentFleetPanel** uses **`useOrgAgentsQuery`**, **`useAgentSubmissionCounts7dQuery`**, and **`useAgentSubmissionsLatestBatchQuery`** (`fetchLatestSubmissionsForAgentIds` / **`fetchLatestSubmissionForAgent`** in **`src/lib/data/agent-submissions-latest.ts`**) with Query **`signal`**; scan polling calls the same lib helper (no in-component **`supabase.from`** for submissions).

**Migrated (earlier):** **SavedReportsLibrary** uses **`useQuery`** / **`useMutation`** with **`queryKeys.savedReports.packages`**. **Fleet Command** uses **`useFleetCommandQuery`**. **FirewallLinker** / **FirewallLinkPicker** use **`invalidateFleetRelatedQueries`**. **`invalidateOrgScopedQueries`** after org purge and customer delete (includes **`queryKeys.org.assessmentSnapshots`**). **TenantDashboard** (cloud) uses **`useOrgAssessmentSnapshotsQuery`**; **AssessmentHistory** calls **`invalidateOrgAssessmentSnapshots`** after cloud save/delete/rename/clear-all.

**Management drawer:** PSA / service-key **presence flags** use **`useOrgPsaIntegrationFlagsQuery`** (`queryKeys.org.psaIntegrationFlags`); **`invalidateQueries`** when a PSA setup modal closes. **Data governance** retention copy uses **`useOrgSubmissionRetentionQuery`** (`queryKeys.org.submissionRetention`). **Org cloud purge** uses **`useOrgCloudPurgeMutation`** ([`src/hooks/queries/use-org-cloud-purge-mutation.ts`](../../src/hooks/queries/use-org-cloud-purge-mutation.ts)), which calls **`purgeOrgCloudData`** and **`invalidateOrgScopedQueries`** (including **`clientPortalPreviewAll`** + **`regulatoryDigest.recent`**). **Client View** preview uses **`useManagementDrawerClientPreviewQuery`**; **regulatory digest** uses **`useRegulatoryDigestQuery`**.

**Customer Management:** directory rows (assessments + tenants + agents + portal slugs) use **`useCustomerDirectoryQuery`** (`queryKeys.org.customerDirectory`); **`fetchCustomerDirectory`** lives in **`src/lib/customer-directory.ts`**. **invalidateQueries** after customer delete and after **PortalConfigurator** save.

**Fleet compliance context:** Default **country** + **sector** live on **`central_tenants`** (**`compliance_country`**, **`compliance_environment`**) and **`agent_customer_compliance_environment`** (same columns, keyed by agent bucket). Per-device overrides: **`central_firewalls`** / **`agents`** **`compliance_country`**, **`compliance_state`**, and optional **`map_latitude`** / **`map_longitude`** (WGS84 for Fleet Map pins). **`FleetFirewall.customerComplianceCountry`** + **`fetchFleetBundle`** / **`fleetEffectiveComplianceCountry`** merge defaults for UI and chips. **Fleet Command** customer header saves **`persistFleetCustomerCompliance`**; firewall panel saves **`persistFleetFirewallCompliance`**. Legacy per-row **`compliance_environment`** on firewalls/agents remains a read fallback. RPC **`update_agent_compliance_context`** updates agent country/state/map coordinates. **Assess** **`fleetContext`** merges device and customer defaults in **`Index.tsx`**. **`fetchLinkedCentralFirewallCompliance`** + **`mergeLinkedCentralCustomerContext`** (**`src/lib/linked-firewall-compliance.ts`**) mirror the same merge for **FirewallLinkPicker** (linked row + pre-link preview) and **UploadSection** branding when a config is linked via **`firewall_config_links`**. **`fetchCustomerDirectory`** aggregates firewall countries per customer (**`—`** / single / **Multiple (...)**).

**Search debouncing:** **CustomerManagement** filters the in-memory customer list using **`useDebouncedValue`** (`src/hooks/use-debounced-value.ts`, 300ms) so typing in search does not thrash heavy **`useMemo`** work on large lists. **FleetCommand** firewall search uses the same hook (300ms). **AgentFleetPanel** uses chip filters only (no text search).

**Fetch cancellation:** **AgentFleetPanel** uses Query **`signal`** for agents, 7d counts, and batched latest **`agent_submissions`** (**`useAgentSubmissionsLatestBatchQuery`**). **`useScheduledReportsQuery`**, **`useMspSetupStatusQuery`**, **`usePortalTenantBootstrapQuery`**, and **`useOrgAgentsQuery`** pass **`signal`** via **`supabaseWithAbort`**. Same pattern for **`useCustomerDirectoryQuery`**, **`useOrgTeamRosterQuery`**, **`useOrgPsaIntegrationFlagsQuery`**, **`useOrgSubmissionRetentionQuery`**, **`usePasskeysQuery`**, **`useSeTeamsQuery`**, **`useHealthChecksQuery`** ([`src/lib/supabase-with-abort.ts`](../../src/lib/supabase-with-abort.ts)).

**Done definition (components / pages, March–April 2026):** User-initiated **`fetch`** from **`src/components`** and **`src/pages`** passes **`AbortSignal`** from TanStack **`queryFn({ signal })`**, from **`useAbortableInFlight()`** ([`src/hooks/use-abortable-in-flight.ts`](../../src/hooks/use-abortable-in-flight.ts)), from an effect-scoped **`AbortController`**, or (for **`ClientPortal`** slug / anon **`portal-data`** load) a generation guard plus **`AbortController`** on the edge **`fetch`**. **`streamConfigParse`** / **`streamChat`** accept optional **`signal`** and merge with timeouts via **`mergeAbortSignals`** ([`src/lib/merge-abort-signals.ts`](../../src/lib/merge-abort-signals.ts)); **`use-report-generation`** aborts on unmount. **Intentional non-cancellation:** fire-and-forget **`fetch`** (e.g. **[`product-telemetry.ts`](../../src/lib/product-telemetry.ts)** webhook POST). **Lib helpers outside that boundary** (PDF asset **`fetch`** in **`se-health-check-pdfmake*`**, **`geo-cve`**, **`share-report`**, **`sophos-central`**) may add **`signal`** when refactored. **SE health-check guest Central:** **`callGuestCentral`** accepts optional **`signal`**; **`use-health-check-inner-state`** uses effect **`AbortController`** for firewall licence load and a superseded-connect chain abort + generation guard for **`connectCentral`** (see **`guest-central-api.ts`**).
